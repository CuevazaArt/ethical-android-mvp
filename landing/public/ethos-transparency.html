Transparency & Limits

An honest summary of what Ethos Kernel does, what it does not do, and where the known limits are. Updated April 2026.

🚨 This is a research prototype. Do not use as a standalone safety boundary for high-stakes decisions. See full caveats below.

Safety mechanisms in place

🔒
MalAbs lexical gate
Unicode-normalised substring filter. Always runs before LLM input. Cannot be disabled by operators in production mode.
🧠
Semantic gate (optional)
KERNEL_SEMANTIC_CHAT_GATE=1 adds embedding similarity checks. Hash fallback when Ollama is unreachable.
📋
Immutable constitution (L0)
8 core ethical principles in PreloadedBuffer. Cannot be overridden by operator flags or DAO votes in the current scope.
🔗
Audit trail (optional)
Append-only JSONL log of safety blocks with hash-linked lines. KERNEL_AUDIT_CHAIN_PATH enables it.
⚡
Env validation (strict mode)
KERNEL_ENV_VALIDATION=strict (default) fails fast on incompatible flag combinations at startup.
🧩
Perception clamping
LLM perception JSON is validated and clamped to [0,1] regardless of model output.

Known limits and caveats

Limit Detail
Lexical filter bypass The MalAbs substring list does not understand paraphrase, homoglyphs, or leetspeak variations. Semantic gate reduces but does not eliminate bypass risk.
LLM perception trust A compromised or prompt-injected LLM can push risk/hostility/calm values within [0,1] range coherently. Clamping prevents out-of-range values but not adversarial coherent manipulation.
Ethical weights are priors Mixture weights (utilitarian / deontological / virtue) are engineering-informed priors, not validated against empirical human moral judgement data.
DAO is a simulation MockDAO is an in-process simulation (quadratic voting, audit strings). There is no deployed on-chain product. It should not be treated as tamper-proof consensus.
Sync LLM in async context LLM calls run in worker threads. Async cancellation after timeout does not guarantee LLM request termination. Known issue tracked.
Narrative memory drift Long sessions accumulate narrative fragments. No automatic compaction; memory window is bounded but not yet adaptive.
No hardware This is a pure software behavioral validator. It does not control physical actuators or sensors directly.

Research prototype disclaimer

📚 This project is published as open research. Its purpose is to explore how explicit, auditable ethical reasoning can be implemented and tested in software — not to provide a production safety system.

The kernel implements a weighted mixture scorer (not full Bayesian updating) over three stylized ethical hypotheses. The names BayesianEngine and related env vars refer to this mixture layer. See ADR 0009 for the naming decision.

The bibliography (104 academic references across 14 disciplines) underpins the design decisions. It is maintained in BIBLIOGRAPHY.md.

For the full limits and bottlenecks inventory, see WEAKNESSES_AND_BOTTLENECKS.md.
