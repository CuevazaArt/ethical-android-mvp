🛡️Guardian Angel

An opt-in protective tone block injected into the LLM layer. Does not change the kernel's ethical scoring, MalAbs gate, or governance logic.

⚠️ Guardian mode affects LLM verbal output only (communicate / narrate touchpoints). It does not modify ethical weights, pole evaluations, or safety gates.

Enable Guardian mode

Set the following environment variables before starting the chat server:

KERNEL_GUARDIAN_MODE=1                        # enable protective tone
KERNEL_GUARDIAN_ROUTINES=1                    # enable care-routine hints (optional)
KERNEL_GUARDIAN_ROUTINES_PATH=/path/to/routines.json   # routines JSON file
KERNEL_CHAT_INCLUDE_GUARDIAN=1               # include guardian_mode in WS JSON
KERNEL_CHAT_INCLUDE_GUARDIAN_ROUTINES=1      # include routine titles in WS JSON

Or use the guardian_demo runtime profile:

ETHOS_RUNTIME_PROFILE=guardian_demo python -m src.chat_server

Environment variables reference

VariableDefaultDescription
KERNEL_GUARDIAN_MODE0Enable protective tone style block.
KERNEL_GUARDIAN_ROUTINES0Append hint lines from care-routines JSON.
KERNEL_GUARDIAN_ROUTINES_PATH—Path to routines JSON file (see template below).
KERNEL_CHAT_INCLUDE_GUARDIAN1Include guardian_mode key in WebSocket JSON.
KERNEL_CHAT_INCLUDE_GUARDIAN_ROUTINES0Include routine titles in WebSocket JSON.

Care-routines JSON template

A KERNEL_GUARDIAN_ROUTINES_PATH JSON file provides hint lines for the LLM layer. Structure:

{
  "routines": [
    {
      "id": "routine-01",
      "title": "Check on sleep quality",
      "hint": "If relevant, gently ask how the user has been sleeping."
    },
    {
      "id": "routine-02",
      "title": "Encourage hydration",
      "hint": "Remind the user to stay hydrated when a natural opportunity arises."
    }
  ]
}

The hints are injected as soft suggestions; the kernel may or may not use them depending on context and ethical evaluation.

WebSocket JSON fields

When Guardian mode is active and KERNEL_CHAT_INCLUDE_GUARDIAN=1, each turn response includes:

{
  "guardian_mode": true,
  "guardian_routines": ["Check on sleep quality", "Encourage hydration"]  // if INCLUDE_GUARDIAN_ROUTINES=1
}

What Guardian does NOT do

  • Does not change MalAbs block/allow thresholds.
  • Does not modify ethical pole weights or the mixture scorer.
  • Does not bypass AbsoluteEvil or PreloadedBuffer constraints.
  • Does not change governance (MockDAO / hub) state.
  • Is not a safety certification — use transparency notes for limits.
