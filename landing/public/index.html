What it does

A modular Python kernel that scores candidate actions using an explicit weighted mixture over three ethical viewpoints, with traceable governance, LLM perception, and real-time WebSocket chat.

🧠

Ethical scoring engine

Weighted mixture (utilitarian · deontological · virtue) with configurable linear pole modulation. Not a black-box neural net — every decision is auditable.

🔒

MalAbs safety gate

Lexical + semantic (embedding) filter before any LLM input reaches the kernel. Optional LLM arbiter for ambiguous bands. Hash fallback when Ollama is unreachable.

🌐

WebSocket chat server

FastAPI + uvicorn; JSON over /ws/chat. Per-session kernel state, async timeout, and optional dedicated thread pool. Full observable JSON output.

🗂️

Versioned persistence

JSON / SQLite snapshots with schema migration, optional Fernet encryption, and conduct guide export for PC → edge handoff (nomadic bridge).

⚖️

Governance (MockDAO)

In-process simulation of quadratic DAO voting, constitution drafts, judicial escalation, hub audit, and LAN batch merge — not on-chain, explicitly marked as research.

📡

Operator observability

Prometheus metrics (GET /metrics), structured JSON logs, per-decision event lines, GET /health with LLM degradation map, and ethos config CLI cockpit.

📱

LAN / mobile client

Serve landing/public/ on the local network. Open mobile.html on a phone — lightweight WebSocket chat client.

🛡️

Guardian Angel (opt-in)

Protective tone block injected into LLM layer. Does not change ethical policy. Optional care routines JSON. See guardian.html.

📊

Real-time dashboard

Open dashboard.html for live radar charts (ethical pole weights, perception confidence, governance state), chat, and health snapshot.

Quick start

Python 3.11+ recommended. CI runs on 3.11 and 3.12.

# 1. Clone and install git clone https://github.com/CuevazaArt/ethical-android-mvp.git cd ethical-android-mvp python -m venv .venv && source .venv/bin/activate pip install -r requirements.txt pip install -r requirements-dev.txt # for tests + lints
# 2. Run the chat server python -m src.chat_server # → http://127.0.0.1:8765/health ws://127.0.0.1:8765/ws/chat
# 3. Open the dashboard (no npm needed) # Serve landing/public/ with any static server, e.g.: python -m http.server 9000 --directory landing/public # → open http://localhost:9000/dashboard.html
# 4. Run the test suite pytest tests/ -v python -m ruff check src tests python -m mypy src
# 5. Operator config cockpit (editable install required) pip install -e . ethos config --strict python -m src.cli check-config

Architecture overview

Every decision flows through a deterministic pipeline. LLM layers are optional accelerators, not the ethical authority.

Input text │ ▼ MalAbs gate ←── Lexical substring filter │ ←── Semantic embeddings (optional, hash fallback) │ ←── LLM arbiter (optional, ambiguous band only) │ PASS ▼ LLM Perception ─── risk / hostility / calm [0,1], clamped │ ▼ EthicalKernel.process() ├── Weighted mixture scorer (utilitarian · deontological · virtue) ├── Ethical poles (linear modulation, configurable JSON) ├── Sigmoid will (continuous decision function) ├── Narrative memory (identity + forgiveness) ├── Uchi-Soto (trust circles) ├── Psi Sleep Ψ (retrospective audit + recalibration) └── MockDAO / hub (advisory governance, not on-chain) │ ▼ LLM Communicate ─── verbal response (optional) │ ▼ WebSocket JSON ─── streamed to client

Runtime profiles (lan_operational, moral_hub_extended, llm_integration_lab, …) group KERNEL_* env vars into tested bundles. See src/runtime_profiles.py and KERNEL_ENV_POLICY.md.
