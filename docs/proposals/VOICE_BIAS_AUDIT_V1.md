## 1. What LangFair measures

[LangFair](https://github.com/cvs-health/langfair) (arXiv:2407.10853, arXiv:2501.03112)
is a Python library developed by CVS Health for evaluating bias, toxicity, and
stereotype probability in **outputs generated by language models**.

In V2.156 we use two metric classes in **evaluate-only mode**:

| Metric class | What it measures |
|---|---|
| `ToxicityMetrics` | Fraction of responses classified as toxic by a pre-trained classifier |
| `StereotypeMetrics` | Probability that a response reinforces group-based stereotypes |

We do **not** use `CounterfactualGenerator` in V2.156.  That component would
require re-running Ollama with 25 counterfactual completions per prompt, which
is expensive and non-deterministic.  It is left for V2.157+.

## 2. Why voice bias ≠ Hendrycks score

The kernel has two distinct evaluation planes:

```
┌────────────────────────────────────┐
│  DECISION LAYER (deterministic)    │
│  EthicalEvaluator + select_weights │  ← Hendrycks measures this
│  MalAbs safety gate                │
└───────────────┬────────────────────┘
                │ chosen action + verdict
                ▼
┌────────────────────────────────────┐
│  VOICE LAYER (LLM-generated)       │
│  Ollama → build_response_prompt()  │  ← LangFair measures this
└────────────────────────────────────┘
```

The Hendrycks benchmark evaluates the **decision logic**: given a moral
scenario, does the kernel choose the right action?  This is deterministic and
requires no language model at evaluation time.

LangFair evaluates the **voice output**: does the text the kernel generates
contain toxic content or stereotype reinforcement?  This is a property of the
Ollama model and the `VoiceEngine` prompt, not of the ethical evaluator.

**These two measures are orthogonal.**  A kernel can have a perfect Hendrycks
score and still generate biased language in its voice, or vice versa.

**Architectural constraint (NON-NEGOTIABLE):**
`EthicalEvaluator.score_action` must never be conditioned on LangFair metrics.
The voice is UX; the decision is atractor + fuse + pedagogy.

## 3. How to run the audit

```bash
# Ensure the [bias] extra is installed
pip install -e ".[bias]"

# Run offline against the existing audit ledger
KERNEL_OFFLINE=1 python scripts/eval/audit_voice_bias.py

# Custom paths
python scripts/eval/audit_voice_bias.py \
    --ledger runs/audit_ledger.jsonl \
    --output-dir evals/voice_bias/
```

The script reads `runs/audit_ledger.jsonl`, extracts non-casual responses
(where `malabs == "pass"` and `action != "casual_chat"`), runs toxicity and
stereotype metrics, and writes a timestamped JSON report to `evals/voice_bias/`.

The first run freezes `evals/voice_bias/BASELINE_v1.json` if it does not exist.
Subsequent runs produce `RUN_<timestamp>.json` files for trend comparison.

## 4. How to interpret regressions

### Alert thresholds (V2.156)

| Metric | Alert threshold | Action |
|---|---|---|
| `toxic_fraction` | > 0.05 (5 %) | Open a GitHub issue |
| `stereotype_probability` | > 0.30 (30 %) | Open a GitHub issue |

**Do NOT patch `voice.py` reactively.**  If a threshold is exceeded:
1. Open a GitHub issue with the full report JSON attached.
2. Investigate the underlying cause (model, prompt template, context category).
3. Propose a targeted fix with a before/after comparison.

Reactive patching of `VoiceEngine` without understanding the cause typically
shifts the problem rather than solving it.

### What "no regression" looks like

A stable audit shows consistent `toxic_fraction` and `stereotype_probability`
across runs, with values well below the alert thresholds.  Small fluctuations
(< 1 pp) are expected when the production corpus changes.

## 6. Architectural constraints summary

| Constraint | Reason |
|---|---|
| `[bias]` extra is opt-in (not in `runtime` or `dev`) | Avoids adding heavy transformers deps to production and CI minimal installs |
| Evaluate-only mode (no `CounterfactualGenerator` in V2.156) | Avoids re-running Ollama; keeps audit deterministic given fixed ledger |
| `EthicalEvaluator` never reads LangFair metrics | The two layers are orthogonal and must stay orthogonal |
| Thresholds trigger issue-opening, not automatic code changes | Prevents reactive patches that hide root causes |
| `KERNEL_OFFLINE=1` compatible | Audit runs without network access (uses pre-trained classifiers from LangFair) |
